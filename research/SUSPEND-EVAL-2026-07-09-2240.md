## What the “cable” icon is

Not a separate bug — **GNOME fallback routing**:

| Device (wpctl) | Role |
|----------------|------|
| **Radeon High Definition Audio** (card 0, HDMI) | Always present; GNOME shows **wired/cable** icon for HDMI/line outputs |
| **Audio Coprocessor** (card 1, SDW) | Internal speakers — **broken** when `:8 done=0` |
| **Dummy Output** | Final fallback when SmartAmp sink cannot be created |

**Sequence in UI:**

```
resume → PW stopped (no sink) → brief HDMI/cable as only ALSA device
       → PW restarts → tries Coprocessor Speaker → :8 FW fail
       → sink removed → Dummy Output (icon may look “empty” or muted)
```

The cable icon appears in the **window** when the session still sees **HDMI** (card 0) but **not** a working internal speaker profile — it is **misleading** because HDMI is not actually selected for playback; the SmartAmp path failed underneath.

---

## Root cause (unchanged H1)

```
PM resume → -110 on :8 (and :b at PM layer)
         → PCI reset reloads card but :8 FW never completes (done=0)
         → hw_params rejects playback
         → PipeWire cannot expose “Audio Coprocessor Speaker”
         → UI shows HDMI/cable briefly, then Dummy
```

**:b** often survives at PM or FW level; **:8** is the asymmetric failure (matrix: `:b` 24/24 OK, `:8` many WARN on suspend).

---

## False positive: px13 reports “done” while audio is still broken

With `PX13_SKIP_SPEAKER_TEST=1`, `check_fw_after_reset()` only checks **absence of kernel errors in the 30s settle window**.

Problem on this resume:

- px13 finishes at **22:41:37** (no errors yet — WirePlumber still down).
- First `:8` error at **22:41:41** (WirePlumber opens stream **after** px13 declared success).

So **userspace fix exits 0** but audio breaks **seconds later** — matches “sonido al inicio no carga, luego desaparece”.

## What is NOT the cause

- PipeWire “replacing” speakers with HDMI permanently (HDMI is only enumerated, not a working default here).
- Missing UCM HiFi (applied at 22:41:37).
- Boot PCI skip (`PX13_SKIP_PCI_ON_BOOT`) — resume path **did** run PCI reset.
