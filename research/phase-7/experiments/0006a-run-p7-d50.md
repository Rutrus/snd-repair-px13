## Verdict

**Outcome A confirmed.** Manual `schedule_work(amd_sdw_irq_thread)` when `STAT(1) & manager_mask == 0x4` at +50 ms **unblocks full SoundWire enumeration** and RT721 resume (`ret=0`).

Root cause localised: **IRQ delivery** from ACP block to `acp63_irq_handler` — not manager bringup, not ping processing, not RT721 wait logic.

Hunt witness `PARTIAL` (legacy classifier): requires `irq_handler_enter` for `PASS`; this run had **no** `pci-ps` handler but **yes** `irq_thread_enter` after manual schedule. See `PASS-0006a` in updated hunt classifier.

## Implications

1. **Upstream fix target:** MSI / `pci-ps.c` `acp63_irq_handler` path for `ACP_SDW1_STAT` on `ACP_EXTERNAL_INTR_STAT1` (instance 1, link 1), not SoundWire manager resume sequence.
2. **0006c not needed** unless reproducing without manual schedule fails intermittently.
3. **Production fix candidates:** ensure ISR runs when STAT1 bit 2 set; or kernel workaround polling STAT&mask post-resume (out of scope for research tree).
