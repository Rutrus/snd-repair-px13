## Four-checkpoint tree (collapses the search space)

Instrument **one boot** post `manager_reset` and mark each checkpoint:

```text
C1  ACP irq_handler entry          (PHASE7 fn=irq_handler_enter)
      ↓
C2  ACP handler return path         (WAKE_THREAD / HANDLED sdw1 / NONE)
      ↓
C3  AMD irq_thread                  (PHASE6 fn=irq_thread_enter)
      ↓
C4  handle_status                   (PHASE6 fn=handle_status)
      ↓
C5  state_change(ATTACHED)          (PHASE6 ctx=sdw fn=state_change … new=ATTACHED)
```

### Decision table

| C1 | C2 | C3 | C4 | C5 | Likely break class |
|----|----|----|----|-----|-------------------|
| no | — | — | — | — | Delivery before handler (legacy IRQ / MSI / mask) — see Phase 8.1 |
| yes | no WAKE/HANDLED sdw1 | — | — | — | Handler filters event / wrong STAT branch |
| yes | WAKE_THREAD missing | no | — | — | Scheduler / threaded IRQ wake |
| yes | * | no | — | — | Thread not scheduled / blocked |
| yes | * | yes | no | — | `amd_manager` status decode / queue_work |
| yes | * | yes | yes | no | SoundWire core state machine |
| yes | * | yes | yes | yes | Downstream — back to codec (`tas_update_status` / `io_init`) |

Note: PHASE6 AMD probes use `resume=0` during normal runtime **and** after system resume — do **not** filter on `resume=1` for C3–C4.
